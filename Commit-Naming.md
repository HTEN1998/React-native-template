<type>(optional-scope): <short summary>  
(optional detailed description)  
(optional footer such as issue/ticket link)

| Type | Purpose |
|----------|----------|
| `feat`   | Adding new feature |
| `fix`    | Fixing a bug   |
| `docs`    | Documentation changes  |
| `style`    | Code formatting or UI-only changes (no logic change)   |
| `refactor`    | Code refactoring without feature change   |
| `pref`    | Performance improvement   |
| `test`    | Adding/modifying tests   |
| `config`    | Build tasks, config updates, dependency updates   |
| `ci`    | CICD pipeline changes   |


For example

feat(api): integrate Stripe payment endpoint

feat(api): add request to fetch user session details

fix(api): correct payload for event creation endpoint

refactor(api): extract axios service methods into helpers

chore(api): update axios interceptor for token refresh

feat(calendar-api): integrate Google Calendar event creation
